## 根因分析

### 根因

**LWIP 网络资源严重泄漏 + 堆内存碎片化双重打击，导致 CoAP 发送过程中内存分配失败，触发 ASSERT 死机。**

### 证据链

1. **异常类型 ASSERT**：`ec_start_flag=0xA2A0A1A3`, `ec_assert_flag=0xB2B0B1B3`, `R0=0`（malloc 返回 NULL）
2. **PC 在 pvPortMallocEC**：`configASSERT(pvReturn != 0)` 断言失败，R0=0 证实分配返回 NULL
3. **LR 指向 coap_socket_connect_tcp1**：崩溃发生在 CoAP TCP 连接建立过程中
4. **LWIP 全部 17 个内存池 100% 耗尽**：TCP_PCB(7/7)、NETCONN(9/9)、PBUF(8/8) 等全满，新连接无法分配任何网络资源
5. **堆严重碎片化**：已用 44.7%，最大空闲块仅 20 字节，无法满足连续分配请求
6. **trace_node 满 (128/128)**：分配/释放活动频繁，碎片化随时间加剧
7. **CmsTask 大量未释放块**：12 blocks 共 19204 bytes（5120+4900+3352+...），可能未及时释放
8. **栈溢出排除**：coapSend 任务栈使用率仅 25%，25 个任务均无栈溢出
9. **DTLS 密钥运算**：mbedTLS `rsa_check_privkey` 在调用链中，密钥运算消耗大量 heap 内存

### 致命链条

```
长期挂测 900+ 次
  → LWIP 网络资源（TCP_PCB/NETCONN/PBUF）累积未释放（泄漏）
    → 全部 17 个 memp 池 100% 耗尽
      → CmsTask/coapSen 反复 malloc/free → 堆碎片化加剧
        → 最大连续空闲块缩小至 20 字节
          → CoAP 发送任务调用 coap_task_send_process
            → coap_socket_connect_tcp1 尝试建立 TCP 连接
              → 底层 lwip_socket/connect 需要分配 TCP_PCB/NETCONN/PBUF → 全部耗尽
                → 同时 heap 也无法分配连续内存
                  → pvPortMallocEC 返回 NULL → ASSERT 触发 → 死机
```

### 问题复现路径

| 项目 | 内容 |
|------|------|
| **前置条件** | 设备网络注册成功 + PDP 已激活 + CoAP 客户端功能开启（DTLS 加密） |
| **必要状态** | LWIP memp 池利用率接近 100% + 堆最大空闲块 < 1024 字节 |
| **操作步骤** | 1. 设备开机，网络附着成功<br>2. 持续进行 CoAP 收发操作（含 CmsTask 业务交互）<br>3. LWIP 资源逐步泄漏累积至全部耗尽，堆碎片化同步加剧<br>4. 经过约 900+ 次循环后达到临界点<br>5. 下一次 CoAP 发送请求触发 TCP 连接 → 内存分配失败 → ASSERT 死机 |
| **复现概率** | 低概率，约 900+ 次挂测后出现（与连接次数和分配模式正相关） |
| **验证方法** | ① 在 LWIP memp 分配/释放路径添加计数器，长期运行后检查各池利用率<br>② 复现后抓取 dump 确认相同 ASSERT + memp 耗尽 + 堆碎片化模式 |

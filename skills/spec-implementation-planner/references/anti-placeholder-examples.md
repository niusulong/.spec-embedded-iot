# 反空话正反对照（Anti-Placeholder Examples）

> 配合 SKILL Step 5「反空话校验」。空话任务 = 没有可勾选的验收判据；最危险的是**变体空话**——看起来具体，实则无法判定"做完没有"。下面每组：左 ❌ 空话，右 ✅ 可验证，并说明右边补了什么验收信号。

## 判据

一条任务合格，当且仅当：**任何人都能据"验收点"判定它做没做完，且"验证方式"能真的跑出来**。否则继续拆细或写具体。

---

## 对照组

### 1. 重拨逻辑
- ❌ **实现 PPP 重拨逻辑**
  - 为什么不合格：没说触发条件、重拨间隔、次数上限、成功判据——"实现完"无法判定。
- ✅ **PPP 断开后按 NV 配置的间隔重拨，最多 N 次，成功即停**
  - 验收点：断网后日志出现 redial 序列；拨号成功恢复数据链路；连续 N 次失败后停止并上报
  - 验证方式：AT 命令断网 + 抓包看重连时序；日志计数 redial 次数 ≤ N

### 2. 异常处理
- ❌ **处理拨号失败异常**
  - 为什么不合格："处理"是空动词，不知道处理成什么样。
- ✅ **拨号连续失败 ≥3 次后进入降节能态并上报 EVT_NET_DOWN**
  - 验收点：注入拨号失败后 30s 内进入降节能态；事件 EVT_NET_DOWN 上报一次
  - 验证方式：故障注入 + 事件日志核对

### 3. 模块对接
- ❌ **与 TCP 模块对接**
  - 为什么不合格：没说对接哪个接口、对接后达成什么效果。
- ✅ **实现 tcp_server_rebuild()，拨号恢复后自动重建 TCP 服务端监听，对上层透明**
  - 验收点：恢复后上层无需重连即可收发；监听 socket 重建成功
  - 验证方式：断网恢复 + TCP 收发测试；抓包确认重新 SYN/ACK

### 4. 电源管理
- ❌ **完善电源管理功能**
  - 为什么不合格："完善"无边界，验收无法收敛。
- ✅ **新增 pm_enter_sleep(mode) 接口，进入休眠后整机电流 < XmA、唤醒恢复 < Yms**
  - 验收点：万用表测电流达标；唤醒后任务恢复正常调度
  - 验证方式：功耗仪测电流；示波器测唤醒时序

### 5. 测试任务
- ❌ **编写相关测试**
  - 为什么不合格：没说测什么、测到什么算过。
- ✅ **test_ppp_redial_interval()：断网后断言重拨间隔 == NV 配置值、重拨计数正确**
  - 验收点：单测通过，覆盖正常重连与达上限停止两条路径
  - 验证方式：运行单测套件 PASS

## 常见空话词（标题/验收点出现即判不合格）

实现 XX 逻辑 · 处理 XX 异常 · 完善 XX 功能 · 与 XX 对接 · 优化 XX · 加固 XX · 类似任务 N · 后续支持 · 视情况处理 · 做好兼容

这些词的共同点：**没有可观测的完成态**。改写时补上"触发条件 + 量化指标 + 成功判据"。

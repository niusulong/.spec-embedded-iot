# 对比分析方法指南

**适用场景**：只有同时提供正常和异常两组日志时才使用对比分析。

## 对比分析步骤

### Step 1：建立对比基准

| 对比维度 | 分析内容 |
|----------|----------|
| **时间维度** | 日志时间戳差异、耗时差异 |
| **状态维度** | 状态机状态、变量值、标志位 |
| **流程维度** | 函数调用顺序、代码执行路径 |
| **数据维度** | 输入参数、返回值、缓冲区内容 |
| **环境维度** | 网络状态、配置参数、资源占用 |

### Step 2：执行差异对比

**正常日志示例**：
```
[10:00:01.123] state=IDLE
[10:00:01.456] func_A() called, param=100
[10:00:01.789] func_B() returned, ret=0
[10:00:02.012] state=CONNECTED
```

**异常日志示例**：
```
[10:05:01.123] state=IDLE
[10:05:01.456] func_A() called, param=100
[10:05:01.789] func_B() returned, ret=-1  ← 差异点
[10:05:02.012] state=ERROR                 ← 差异点
```

### Step 3：差异点分析

使用表格整理差异点：

| 时间点 | 正常日志值 | 异常日志值 | 差异说明 |
|--------|-----------|-----------|----------|
| T1 | param=100 | param=100 | 相同 |
| T2 | ret=0 | ret=-1 | **关键差异** |
| T3 | state=CONNECTED | state=ERROR | 后续差异 |

### Step 4：追溯差异根因

```
差异点：ret=-1（异常）vs ret=0（正常）

追溯分析：
1. ret=-1 是什么错误？
   → 查找错误码定义：-1 = ERROR_TIMEOUT

2. 为什么正常时 ret=0，异常时 ret=-1？
   → 检查输入参数：param 相同，排除参数差异
   → 检查环境状态：正常时网络良好，异常时网络波动

3. 为什么网络波动导致超时？
   → 检查超时配置：timeout=5ms，配置过短

根本原因：超时配置过短，网络波动时容易超时
```

## 完整示例

### 问题场景

HTTPS请求偶现失败，成功率约80%。

### 对比环境

| 对比维度 | 正常日志 | 异常日志 |
|----------|----------|----------|
| 日志文件 | success.log | fail.log |
| 时间范围 | 10:00-10:05 | 10:10-10:15 |
| 场景描述 | HTTPS请求成功 | HTTPS请求失败 |

### 关键差异点

| 时间点 | 正常日志值 | 异常日志值 | 差异说明 |
|--------|-----------|-----------|----------|
| T1 | msglen=3200 | msglen=5646 | 消息长度不同 |
| T2 | maxBufLen=4400 | maxBufLen=4400 | 相同 |
| T3 | ret=0 | ret=-0x7200 | **关键差异** |

### 差异分析

1. 异常日志中消息长度(5646)超过缓冲区大小(4400)
2. 错误码 -0x7200 对应 `MBEDTLS_ERR_SSL_BUFFER_TOO_SMALL`
3. 正常日志中消息长度(3200)在缓冲区范围内

### 根本原因

SSL缓冲区配置不足（4400字节），部分大消息无法处理。

- **release：插件版本 1.1.0 → 1.2.0**（知识库检索本轮增强：混合索引 + 审查优化 Tier 0–3 后的发版）。
- **chore(knowledge/ASR1603)：回填工作项ID + 纳入 rti_switchout bug 文档**——4 条历史文档补「工作项 ID = NA」（与 EC626 一致），新文档 `ASR1603_NA_rti_switchout_null_deref.md` 纳入；meta 标记 `schema_version=1`，重建 bug-solutions 向量索引，verify 通过。
- **feat(archiver/retrieval)：混合索引（摘要 + 正文分块）**
  - bug-solutions / requirement-solutions 新增 `index_body:true`：除摘要向量外，对其 `.md` 正文分块建 `body` 向量，补召回（日志/寄存器/AT 命令等正文细节可语义检索）。
  - 摘要向量 ID `{plat}/{file}`（向后兼容），正文块 ID `{plat}/{file}/chunk_{n}`。
  - 空摘要时经正文兜底仍可检索；既无摘要又无正文才警告跳过。
- **feat(search)：按源文件去重 + 命中类型标签**
  - 过取后按 `(platform,file)` 去重，避免 top-N 被同一文档多块占满。
  - 输出 `[摘要]`/`[正文]` 标签，`[正文]` 显示命中段落标题。
- **docs：新增 `docs/` 文档目录**（功能说明 / 历史修改 / 审查与优化）。
- **review：完成知识库检索架构多角度审查**，归档审查报告 + 优化方案（见 `审查与优化/2026-06-17-知识库检索架构审查报告.md`）。
- **fix(archiver/retrieval)：Tier 0 正确性修复（审查报告 C1–C4，全部已测）**
  - **C1 verify 向量校验复活**：移除不存在的 `COLLECTION_NAME` 导入，改用 `cfg["dest_dir"]` 定位 collection（顺带覆盖 requirement），`except:pass` 改为 warn。已证明检查4代码路径实际执行。
  - **C2 距离度量改 cosine**：所有 `get_or_create_collection` 加 `hnsw:space=cosine`，全量重建；`1-distance` 现为真实 cosine 相似度（相关查询 0.7–0.8，无关查询 ~0.24，区分度明显）。
  - **C3 chunk_markdown 去垃圾块**：大节按 `body` 切分（标题附加首段，不再独立成块）+ 空输入返回 `[]`；bug-solutions 索引 349→347（消除标题垃圾块）。
  - **C4 表格正则严格化**：`TABLE_ROW_RE` 用 `[^|]` 字符类，只匹配 2 列行，3+ 列表格不再被吞列污染（单测：2 列/加粗/list/分隔行均正确）。
- **fix(archiver/retrieval)：Tier 1 重要可靠性修复（审查报告 H1/H2/H3/H5/H6，全部已测）**
  - **H1 相似度阈值**：`embed_search` 加 `--min-similarity`（cosine，默认 0.0 不过滤；RAG 建议 0.3）。阈值 0.3 下相关查询保留、无关查询被过滤。
  - **H2 verify 反向僵尸校验**：检查4新增反向扫描，检出当前平台 meta 已不存在的 `[僵尸向量]`（端到端验证：注入假向量→检出→清理→恢复 OK）。
  - **H3 向量 id 与文件名解耦**：向量 id 基础从 `file` 改为 `entry_name`（`{plat}/{entry_name}[ /chunk_N]`），根治文件名迁移（加平台前缀）导致的 id 漂移；verify 检查4同步。已全量重建。
  - **H5 库函数抛异常**：`get_doc_type_config`/`get_collection_config` 改抛 `ValueError`（替代 `sys.exit`），便于作为库复用与测试；CLI 入口捕获转友好报错。
  - **H6 静默 except 降级**：摘要提取失败、`extract_summary --type` 加载失败等隐藏真实问题的 `except:pass` 改为 `[warn]` 到 stderr。
- **refactor(common)：Tier 2 架构/可维护性（审查报告 H4/H7/M5/M2/M3，全部已测）**
  - **H4 配置校验 + 深度合并**：`load_config` 深度合并默认值（缺键回退，防静默瘫痪）+ `_validate_config`（必需键、summary collection↔doc_type 关系、version 告警）；顺带消除 doc_type↔collection 隐式耦合（加载期断言）。
  - **H7 缓存安全**：`load_config` 返回 deepcopy 防污染；embedding 单例双检锁；新增 `reload_config()`。
  - **M5 统一正则**：抽 `SUMMARY_HEADING_RE` 常量；修复 ensure 的 `\s*$` 严格锚点导致带后缀标题（如「## 结构化摘要（专项）」）不注入 NA 的不一致。
  - **M2/M3 去重+过取**：去重 key 加 collection 维度；过取因子 `top*5`→`top*10`。
- **chore/test：Tier 3 工程化（依赖/分隔符/锁/测试，全部已测）**
  - 声明运行时依赖：`requirements.txt`（chromadb>=1.5、sentence-transformers>=5.0）。
  - doc_id 分隔符统一：`_collect_markdown_chunks` 的 rel_path/sub_path 统一正斜杠，跨平台一致（实测 code-summary 159 块无反斜杠）。
  - **H8 meta schema_version + 文件锁**：`save_meta` 标记 `schema_version=1`，`load_meta` 检查兼容性告警；归档临界区加跨平台独占文件锁（标准库 msvcrt/fcntl，零依赖），防并发 last-write-wins。
  - 补单元测试：`test_archiver.py`（unittest，24 项，覆盖 chunk_markdown/build_summary_text/表格解析/safe_filename/extract_*/ensure/deep_merge/validate）。

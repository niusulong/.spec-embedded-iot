# docs — 文档目录

本目录存放 `spec-embedded-iot` 插件（嵌入式 IoT 开发技能库 & 知识库）的说明文档、历史修改记录与审查优化中间文档。

## 目录结构

```
docs/
├── README.md                 本索引
├── 功能说明.md                spec-knowledge-archiver 功能、架构、命令、配置
├── 历史修改记录.md            按时间倒序的 changelog
├── 功能变更/                 每次新增/重大功能改动的归档
│   └── 2026-06-18-方案与计划技能链路.md       v1.3.0 方案/计划两技能 + 审查优化
└── 审查与优化/
    └── 2026-06-17-知识库检索架构审查报告.md   多角度代码审查 + 分层优化方案
```

## 约定

- **审查与优化/** 子目录按 `{日期}-{主题}.md` 命名，存放每次审查的中间文档与方案，便于回溯决策。
- **功能变更/** 子目录同样按 `{日期}-{主题}.md` 命名，存放每次新增或重大功能改动的归档（背景/设计决策/审查优化/文件清单/后续）。
- 历史修改记录与 git 提交对齐；未提交的工作会明确标注「未提交」。
- 新增审查报告时，在 [历史修改记录.md](./历史修改记录.md) 追加一条指向。
